# Email-уведомления - Краткая справка

## ✅ Реализованные функции

### 1. 🔐 Уведомление о входе в аккаунт
**Кому:** Пользователю, который вошел в систему
**Когда:** При каждом входе в аккаунт
**Содержание:**
- Дата и время входа
- IP-адрес
- Информация о браузере

**Файлы:**
- `users/signals.py` - сигнал на вход
- `templates/emails/login_notification.html` - шаблон письма

---

### 2. 📋 Уведомление о новой заявке
**Кому:** Всем работникам и администраторам
**Когда:** При создании новой заявки клиентом
**Содержание:**
- Номер заявки
- Данные клиента (ФИО, email, телефон)
- Тип заявки
- Описание
- Дата создания

**Файлы:**
- `orders/views.py` (функция `order_create`) - вызов уведомления
- `templates/emails/new_order_notification.html` - шаблон письма

---

### 3. 👤 Уведомление о назначении ответственным
**Кому:** Работнику, назначенному ответственным за заявку
**Когда:** При назначении или изменении ответственного
**Содержание:**
- Номер заявки
- Статус заявки
- Полная информация о клиенте
- Описание проблемы
- Сумма заказа

**Файлы:**
- `orders/views.py` (функция `order_edit`) - проверка назначения
- `templates/emails/order_assigned_notification.html` - шаблон письма

---

### 4. ⚠️ Уведомление о низком остатке
**Кому:** Всем работникам и администраторам
**Когда:** При изменении количества товара (если достигнут минимальный остаток)
**Содержание:**
- Название товара
- Артикул
- Категория
- Текущий остаток
- Минимальный остаток
- Цена

**Файлы:**
- `inventory/signals.py` - сигнал на изменение количества
- `templates/emails/low_stock_notification.html` - шаблон письма

## ⚡ Быстрый старт

### 1. Настройте email (Gmail)

Создайте файл `.env`:
```env
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=ваш-email@gmail.com
EMAIL_HOST_PASSWORD=пароль-приложения-16-символов
DEFAULT_FROM_EMAIL=ваш-email@gmail.com
```

### 2. Получите пароль приложения Gmail

1. https://myaccount.google.com/ → Безопасность
2. Включите двухфакторную аутентификацию
3. Пароли приложений → Создать → Скопируйте пароль

### 3. Проверьте настройки

```python
# В settings.py уже добавлены настройки:
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = os.getenv('EMAIL_HOST', 'smtp.gmail.com')
...
```

### 4. Для тестирования без отправки

В `settings.py` замените:
```python
EMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend'
```

Письма будут выводиться в консоль вместо отправки.

## 📝 Требования

- У пользователей должны быть указаны email адреса
- Для Gmail нужен пароль приложения (не обычный пароль)
- Работает с любым SMTP сервером

## 🔍 Проверка работы

1. **Вход в систему** - войдите в аккаунт, проверьте email
2. **Новая заявка** - создайте заявку от клиента, работники получат письмо
3. **Назначение** - назначьте ответственного, он получит письмо
4. **Низкий остаток** - измените количество товара до минимума, работники получат письмо

## 🎨 Кастомизация

### Изменить шаблоны писем
Отредактируйте файлы в `templates/emails/`

### Изменить получателей
Отредактируйте функции в `core/notifications.py`

### Отключить уведомления
Закомментируйте вызовы в соответствующих файлах или используйте консольный бэкенд
