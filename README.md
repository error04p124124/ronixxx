# Система управления запасами и комплектующими

## ООО НПФ «Роникс-Л»

Информационная система для управления запасами, обработки заявок и учёта комплектующих в сервисном центре.

---

## 📋 Описание проекта

Дипломная работа на тему: **"Проектирование и разработка информационной системы управления запасами и комплектующими в ООО НПФ „Роникс‑Л"**

Система предоставляет полный функционал для:
- Управления складскими запасами
- Обработки заявок от клиентов
- Формирования отчётности
- Управления пользователями с различными ролями доступа

---

## 🏗️ Архитектура проекта

### Технологический стек

- **Backend:** Python 3.11+, Django 5.0+
- **Frontend:** Bootstrap 5, JavaScript (vanilla)
- **База данных:** SQLite (по умолчанию), PostgreSQL (опционально)
- **Дополнительно:** WeasyPrint/ReportLab для PDF, django-tables2, django-crispy-forms

### Структура приложений

```
ronix/
├── ronix_project/          # Основные настройки проекта
├── core/                   # Базовые шаблоны, главная страница, темизация
├── users/                  # Управление пользователями, роли, аутентификация
├── inventory/              # Управление складом и комплектующими
├── orders/                 # Заявки, заказы, чеки
├── reports/                # Генерация отчётов
├── templates/              # HTML шаблоны
├── static/                 # Статические файлы (CSS, JS, изображения)
└── media/                  # Загружаемые файлы
```

## 👥 Роли пользователей

### 1. **Клиент** (`client`)
- Просмотр своих заявок
- Создание новых заявок на обслуживание или заказ комплектующих
- Получение чеков за услуги

### 2. **Работник сервисного центра** (`worker`)
- Просмотр и обработка всех заявок
- Управление запасами и комплектующими
- Отметка заявок как выполненных
- Формирование отчётов

### 3. **Управляющий сервисного центра** (`manager`)
- Полный доступ ко всем функциям системы
- Управление пользователями
- Просмотр аналитики и статистики
- Редактирование складских данных
- Формирование расширенных отчётов

## 🚀 Установка и запуск

### Предварительные требования

- Python 3.11 или выше
- pip (менеджер пакетов Python)
- Git (опционально)

### Шаг 1: Клонирование репозитория

```powershell
cd d:\ronix
```

Если проект в Git:
```powershell
git clone <repository-url>
cd ronix
```

### Шаг 2: Создание виртуального окружения

```powershell
python -m venv venv
.\venv\Scripts\activate
```

### Шаг 3: Установка зависимостей

```powershell
pip install -r requirements.txt
```

### Шаг 4: Настройка окружения

Скопируйте `.env.example` в `.env` и настройте параметры:

```powershell
copy .env.example .env
```

Отредактируйте `.env`:
```
SECRET_KEY=your-secret-key-here
DEBUG=True
DATABASE_URL=sqlite:///db.sqlite3
ALLOWED_HOSTS=localhost,127.0.0.1
```

### Шаг 5: Применение миграций

```powershell
python manage.py makemigrations
python manage.py migrate
```

### Шаг 6: Создание суперпользователя

```powershell
python manage.py createsuperuser
```

Следуйте инструкциям для создания администратора.

### Шаг 7: Загрузка тестовых данных (опционально)

```powershell
python manage.py populate_data
```

Эта команда создаст:
- 3 тестовых пользователя (клиент, работник, управляющий)
- Категории комплектующих
- Образцы комплектующих
- Поставщиков
- Примеры заявок

**Тестовые учётные данные:**
- Клиент: `client` / `client123`
- Работник: `worker` / `worker123`
- Управляющий: `manager` / `manager123`

### Шаг 8: Запуск сервера разработки

```powershell
python manage.py runserver
```

Откройте в браузере: [http://localhost:8000](http://localhost:8000)

## 🎨 Основные функции

### 📦 Управление складом

- **Комплектующие:** Добавление, редактирование, удаление
- **Категории:** Группировка комплектующих по типам
- **Поставщики:** Ведение базы поставщиков
- **Движение товаров:** Отслеживание поступлений, списаний, возвратов
- **Уведомления о низких остатках**

### 📋 Управление заявками

- Создание заявок на обслуживание или заказ комплектующих
- Назначение ответственных исполнителей
- Отслеживание статуса выполнения
- Добавление позиций в заявки
- Автоматический расчёт стоимости

### 🧾 Чеки и отчётность

- Автоматическая генерация чеков при завершении заявки
- Формирование отчётов по складу
- Отчёты по заявкам
- Финансовые отчёты
- Экспорт в PDF (HTML preview)

### 🌓 Темизация

- Светлая тема (по умолчанию)
- Тёмная тема
- Переключатель в навигационной панели
- Сохранение выбора темы

## 🗄️ Модели данных

### Users (Пользователи)
- Расширенная модель User с полями роли, телефона, адреса
- Автоматическое присвоение к группам

### Inventory (Склад)
- **Category:** Категории комплектующих
- **Component:** Комплектующие с артикулами, ценами, остатками
- **Supplier:** Поставщики
- **StockMovement:** История движения товаров

### Orders (Заявки)
- **Order:** Заявки с типами, статусами, суммами
- **OrderItem:** Позиции заявок
- **Receipt:** Чеки об оказанных услугах

## 📊 Миграция на PostgreSQL

Для production использования рекомендуется PostgreSQL.

1. Установите PostgreSQL и создайте базу данных

2. Установите psycopg2:
```powershell
pip install psycopg2-binary
```

3. Измените DATABASE_URL в `.env`:
```
DATABASE_URL=postgresql://username:password@localhost:5432/ronix_db
```

4. Примените миграции:
```powershell
python manage.py migrate
```
